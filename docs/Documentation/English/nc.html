nc

• nc[a,b,c] represents non-commutative multiplication, i.e. it encloses a string of operators that do not (necessarily) commute.

• When PrettyOutput is enabled, nc expressions are output in infix notation with CenterDots.

• nc is a linear operator in all of its arguments: scalar (numeric) quantities are factored out in front of the expression, and distributive law over addition is automatically applied. See sneg command sneglinearoperator.

• nc is associative, for example nc[a,nc[b,c],d]=nc[a,b,c,d]. Note, however, that nc does not have Flat, nor OneIdentity attributes like Times has. This was a design decision for reasons of performance. This implies that pattern matching is different for nc as compared to Times. Note also that nc, being noncommutative multiplication, clearly does not have OrderLess attribute.

• nc[] is taken to be 1.

• nc is idempotent, nc[c]=c.

• nc automatically imposes Pauli rule for fermionic operators: a product of two consecutive equal operator is zero.

• Operator strings are automatically reordered into the  normal form. See ordering and acmt for more details.

• In addition to operators, nc can contain Dirac bras and kets (see sneg commands bra and ket) and VACUUM placeholder. More generally, sneg will leave intact any elements that it does not recognize to have special meaning.

• nc can be used to multiply abstract sums. See sum for more details.

• Remember to use nc instead of Times for multiplication when using Mathematica commands such as Inner and Outer (see sneg commands inner, outer, mcross and VMV). Furthermore, use pow instead of Power to raise operator expressions to some power.

• See also: snegfermionoperators, snegrealconstants, vev, vevwick, dd, onvac, ap, wick.
